Error Prevention

"Even better than good error messages is a careful design which prevents a problem from occurring in the first place. Either eliminate error-prone conditions or check for them and present users with a confirmation option before they commit to the action." - Jakob Nielsen

Error prevention is critical for users since it will save users a lot of time and energy trying to find the right thing.

Accidental errors

When the user mouses over the the two arrows, the pop out slides into the page but doesn't know whether the user actually wants to use it or just accidentally rolled over it. In the case that the user actually wants to visit the page that pops out, it doesn't stay for very long so the user can't work it very well. If the user accidentally rolls over it, it pops out in the middle of the screen so then blocks other aspects of the page. These arrows/pop-outs could be handled in a better way or in a better location. It also scrolls with the page instead of being locked into one place. This causes user errors especially since it is on the side of the page where the scroll is located also.

Doesn't correct inaccurate typing

On the maps page, if a user types in an incorrect location or a location that doesn't exist, there is no error message. Instead, a pop up or something should display communicating to the user that the place is non-existant. Instead nothing appears. Also, instead of allowing the user to finish typing the location, it should give the user a message in the typing space.sp>
