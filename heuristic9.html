Help Users Recognize, Diagnose, and Recover from Errors

"Error messages should be expressed in plain language (no codes), precisely indicate the problem, and constructively suggest a solution." - Jakob Nielsen

Providing error messages and explaining steps that can be taken to solve the errors allows users to easily circumvent their problems and achieve their objectives to the best of the website’s abilities. Not knowing what the problem is can make users confused or frustrated, so it is important to clearly state the problem and help the user recover from the error.

Search suggestions

On the frontpage of the university’s website, there is a search bar which allows for users to search for certain terms related to the school itself or its directory. The university uses the search bar powered by Google. When a user misspells a search term, the search bar suggests another correctly spelled term that is most identical to the misspelling. This allows users to continue with their search uninterrupted.

Sign in error

The university offers a variety of resources for students, but they must first sign in with their university account and password. If a user types in the wrong username or password, an error message will be displayed. The error message does not specify which of the two fields are wrong, but this may be due to security reasons to prevent others from hacking into student accounts.

Lack of map feedback

The campus map has a search bar that users can use to find certain buildings or locations on campus. As the user starts typing, certain names will show up in a list displayed immediately below the search bar. However, if a user types in a location that is not found on campus, no error messages or suggestions show up. Pressing on the magnifying glass search button does not yield any changes to the page whatsoever.

Highlighted text field

In the technological help center, users can submit a form with a description of the problem they are experiencing, if they do not find the answer by the other methods provided. The user must fill out all the text fields in order for the form to be submitted successfully. If one of the sections is not completed, then an error message will show up in red text at the top of the form, and the empty text field will be highlighted in red.
