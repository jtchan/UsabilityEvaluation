Help and Documentation

"Even though it is better if the system can be used without documentation, it may be necessary to provide help and documentation. Any such information should be easy to search, focused on the user's task, list concrete steps to be carried out, and not be too large." - Jakob Nielsen

Although a system may seem better without needing documentation, it can still be helpful for the user to have a source that can help them with their task or where they can find solutions to problems they are experiencing. If such a documentation exists, it should not be cumbersome and slow down a user when they are attempting to complete a task.

Topics of aid

The navigation bar for students and faculty associated with the university has a few helpful links, although they are better organized in the dropdown menu for faculty. These links include a link for safety issues, a link to contact the university, a link to the help center for technology, and a link to an index of the topics covered on the site. All these links benefit the user and can help them acquire more knowledge to solve any kind of problems they are experiencing.

Help center

The first navigation bar on the page has links to the help center for technological related issues. To the left of the page, there is a dropdown menu from which the user can select the category that covers their issue. Below the dropdown menu, there is also a search bar where the user can type in their search query. The main section of the page also contains some of the most asked topics, with a link that takes the user to a page with all of the most asked questions. If the user still cannot find the answer to their issue, then they can use the form located in the main section of the page to submit a description of the problem they have encountered.

A-Z index

The university website offers an alphabetically sorted index of topics that are covered on the site. This provided documentation can help users find the topic they are searching for. While the user can simply click on a letter to navigate to the categories beginning with that letter, there is no simple way to navigate back to the top of the page. If a user wanted to sequentially search for more than one term using the list of letters at the top of the page, they would have to scroll all the way up before beginning their search anew.
