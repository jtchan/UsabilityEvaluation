Recognition Rather than Recall

"Minimize the user's memory load by making objects, actions, and options visible. The user should not have to remember information from one part of the dialogue to another. Instructions for use of the system should be visible or easily retrievable whenever appropriate." -Jakob Nielsen

Human brains are programmed so that it is easier to recognize things instead of having to recall things. Therefore, when a user can simply remember something by looking at it then it allows for quicker navigation through the page.

User doesn't know functionality of page

The main aspect of the homepage is poorly designed as the main image allows for some of it to pop-up. When the user mouses over the image, the bottom half comes up, and it doesn't matter if the mouse is even on the bottom. Also, when the picture pops up there is no arrow or anything for the user to recognize how to drop it down again. Therefore, the user has to recall from previous knowledge how to close that part of the screen, by moving the mouse outside.

User has to recall the page titles

When the drop down appears and the user clicks on the link, there is no way to remember the names of the pages from the dropdown. Also, the navigation disappears so the user can't go back to it unless they go back to the homepage. This makes the user think more since they have to recall the navigation.
